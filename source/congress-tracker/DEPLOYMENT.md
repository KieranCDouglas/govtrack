# CongressWatch — GitHub Pages Deployment Guide

## Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                     GitHub Repository                        │
│                                                              │
│  main branch                    gh-pages branch             │
│  ┌─────────────┐                ┌─────────────────────────┐ │
│  │ Source code │  GitHub        │  Built static files     │ │
│  │ + scripts   │ ─Actions──────▶│  index.html             │ │
│  │ + data.db   │                │  assets/*.js / *.css    │ │
│  └─────────────┘                │  data/members-*.json    │ │
│                                 └─────────────────────────┘ │
└──────────────────────────────────────┬──────────────────────┘
                                       │ served by GitHub Pages
                                       ▼
                            https://yourusername.github.io/repo/
```

**No server required.** The app is entirely static after build:
- Member data is pre-generated as JSON files at build time
- Live vote feeds call **GovTrack.us** directly from the browser (CORS: `*`)
- Per-member vote history calls **GovTrack.us** directly from the browser
- Data auto-refreshes weekly via GitHub Actions

## Step 1 — Create a GitHub Repository

1. Go to [github.com/new](https://github.com/new)
2. Name it something like `congress-tracker` or `congresswatch`
3. Keep it **Public** (required for free GitHub Pages)
4. Do **not** initialize with a README (you'll push existing code)

---

## Step 2 — Push Your Code

From the `congress-tracker/` directory on your machine:

```bash
git init
git remote add origin https://github.com/YOURUSERNAME/YOURREPO.git
git add .
git commit -m "Initial commit"
git push -u origin main
```

**What to include** (everything except these, which are in `.gitignore`):
- ✅ All source files (`client/`, `server/`, `shared/`, `scripts/`)
- ✅ `data.db` — SQLite database (the data source for build-time generation)
- ✅ `seed_data.json`, `policy_analysis.json`, `govtrack_current_members.json`
- ✅ `client/public/data/` — Pre-generated static JSON files
- ✅ `.github/workflows/deploy.yml` — The Actions workflow
- ❌ `node_modules/` — Never commit this
- ❌ `dist/` — Built output, generated by Actions

---

## Step 3 — Configure GitHub Pages

1. In your repository, go to **Settings → Pages**
2. Under **Source**, select: **Deploy from a branch**
3. Branch: **`gh-pages`** / Folder: **`/ (root)`**
4. Click **Save**

The `gh-pages` branch will be created automatically by GitHub Actions on first deploy.

---

## Step 4 — Set the Base Path Variable

This tells Vite where the site will be served from.

1. Go to **Settings → Secrets and variables → Actions → Variables**
2. Click **New repository variable**
3. Name: `VITE_BASE_PATH`
4. Value:
   - If your repo is `https://github.com/kieran/congress-tracker`:
     - Value: `/congress-tracker`
   - If you're using a **custom domain** (e.g. `congresswatch.com`):
     - Value: `/`
   - If your repo is `kieran.github.io` (user/org site):
     - Value: `/`

## Step 6 — Trigger the First Deploy

The workflow runs automatically on every push to `main`. After pushing in Step 2, go to:

**Actions tab → "Build and Deploy to GitHub Pages" → watch it run**

It will:
1. Download fresh Voteview DW-NOMINATE scores
2. Fetch current member roster from GovTrack
3. Generate compass coordinates for all 12,500+ members
4. Build the Vite frontend
5. Deploy to `gh-pages` branch → GitHub Pages

**First run takes ~3–5 minutes.** After that, your site is live at:

```
https://YOURUSERNAME.github.io/YOURREPO/
```

## How Data Stays Fresh

### Automatic weekly refresh (Sundays 2am UTC)
The GitHub Actions workflow runs on a schedule. Each Sunday it:
1. Downloads fresh DW-NOMINATE scores from Voteview (updated quarterly)
2. Refreshes the current member roster from GovTrack
3. Rebuilds and redeploys the static site

### Live data on every page load
Even between weekly rebuilds, the site shows live data:
- **Homepage vote feed** — calls GovTrack directly, updates on every page refresh
- **Member vote history** — calls GovTrack directly when you open a member's profile
- Both work entirely in the user's browser, no server needed

### Manual refresh
You can force a fresh build anytime:
1. Go to **Actions → Build and Deploy to GitHub Pages**
2. Click **Run workflow → Run workflow**

## Custom Domain (Optional)

To use a domain like `congresswatch.yourdomain.com`:

1. Add a `CNAME` file to `client/public/` containing just your domain:
   ```
   congresswatch.yourdomain.com
   ```

2. In your DNS settings, add a CNAME record:
   ```
   congresswatch  CNAME  YOURUSERNAME.github.io
   ```

3. Set `VITE_BASE_PATH` to `/` in your repository variables.

4. In GitHub Settings → Pages, enter your custom domain and enable HTTPS.

## Updating the Compass Scores

The DW-NOMINATE scores are updated by Voteview roughly quarterly as Congress casts more votes. The weekly GitHub Actions job automatically picks up new scores. If you want to update scores immediately:

```bash
# Run locally from the congress-tracker/ directory:
python3 scripts/refresh_data.py
python3 scripts/generate_static_data.py

# Then commit and push the updated data files:
git add data.db seed_data.json client/public/data/
git commit -m "Refresh member data"
git push
```

This will trigger a new GitHub Pages deploy within minutes.

## Troubleshooting

**Site shows 404 after deploy:**
- Check that `gh-pages` branch exists in your repository
- Verify GitHub Pages is configured to deploy from `gh-pages` branch
- Ensure `VITE_BASE_PATH` matches your actual repository URL path

**Data files not loading (network errors in console):**
- Open browser DevTools → Network tab
- Look for failed requests to `/data/members-current.json`
- If the path has your repo name doubled (e.g. `/repo/repo/data/...`), check `VITE_BASE_PATH`

**Members page shows no historical members:**
- The `members-index.json` file (~1.1MB) may still be loading
- It loads lazily only when searching historical members
- Check Network tab for the request

**Live vote feed not updating:**
- GovTrack.us may be temporarily down
- The feed auto-refreshes every 5 minutes; try force-refreshing the page
- Check the browser console for CORS errors (should be none — GovTrack allows `*`)
